## 今日重点判断

1. 先进封装的竞争从“能不能堆”转向“能不能系统验证”：EMIB-T、HBM4e、UCIe、CPO、glass substrate 同时出现，说明下一阶段比拼的是 package channel、PDN、thermal、warpage、test insertion 与客户 tape-out。
2. CPO 的下一关是 HVM 与 field service：Lightmatter 的 OIP 摘要把 known-good optical engines、detachable fiber attachment、assembly、packaging、test、cost、quality、reliability 明确列为 CPO 量产挑战。
3. SI/PI 要同时盯 optical 和 copper：switch-side CPO、224G LR SerDes、260G C2C、UCIe live demo 并不是互斥路线，而是在 rack、tray、package、die-to-die 不同距离上做功耗、延迟、可靠性和可维护性的权衡。

### S 级｜ECTC 2026 与 Intel Foundry EMIB-T / glass substrate / CPO 框架

- 日期：2026-06-05
- 来源：Semiconductor Engineering
- 链接：https://semiengineering.com/packaging-technologies-redefine-ai-and-hpc-scalability-limits-at-ectc-2026/
- 事实摘要：文章复盘 ECTC 2026，称 Intel Foundry 与合作者展示约 20 篇技术论文，覆盖 EMIB-T、HBM4e、64 Gb/s UCIe interface performance、CPO、glass core substrate/TGV、hybrid bonding、warpage、AI/HPC large form factor package 等主题。
- 技术影响：这是 advanced packaging system framework。EMIB-T 用 embedded bridge + TSV 改善 lateral interconnect 与 power delivery；glass substrate 用 TGV、低 warpage 和大尺寸稳定性支撑 AI/HPC 大封装；CPO 则把 optical edge connector、glass fan-out coupler、detachable serviceability 引入封装量产问题。
- 评级依据：S 级。头部 foundry + ECTC 技术场景 + 可验证指标 + 同时命中 2.5D/3D、HBM、UCIe、CPO、glass substrate。
- 可信度/待验证：仍需 Intel 原始 technical paper、客户 tape-out、量产良率、warpage data、HBM4e stack compatibility、UCIe margin 和 package reliability 数据。
- 对用户有什么用：变成封装评估矩阵：bridge loss、via parasitic、PDN impedance、HBM SSN、UCIe eye/jitter、thermal gradient、warpage、test insertion。

### A+ 级｜Intel Foundry advanced packaging portfolio

- 日期：2026-06-08 抓取
- 来源：Intel Foundry
- 链接：https://www.intel.com/content/www/us/en/foundry/packaging.html
- 事实摘要：Intel Foundry advanced packaging 页面把 2D/2.5D/3D packaging、logic-HBM integration、UCIe interconnect、Foveros、EMIB、EMIB-T 与 2030 年 1 trillion transistors in a package 的目标放在一起。
- 技术影响：Intel 正把 foundry packaging portfolio 变成客户架构选择的一部分：chiplet partition、HBM attach、UCIe PHY、known-good-die、bridge topology、PDN 与 thermal/mechanical 约束会更早进入架构期。
- 评级依据：A+ 级。官方路线页面可信度高，且与 ECTC 复盘互相印证；但页面偏能力叙事，不等于客户量产验证。
- 可信度/待验证：需要外部客户采用、封装良率、RDL/bridge pitch、UCIe compliance、热机械可靠性和产能爬坡数据。
- 对用户有什么用：对比 TSMC CoWoS/SoIC/COUPE 与 Intel EMIB/Foveros/EMIB-T 时，要比较 bridge topology、HBM count、UCIe PHY、PDN 和热机械约束。

### A 级｜Lightmatter OIP: Scaling Silicon Photonics HVM for CPO

- 日期：event 2026-06-17
- 来源：Lightmatter
- 链接：https://lightmatter.co/event/optical-interconnect-and-packaging-conference/
- 事实摘要：Lightmatter OIP 页面列出 2026-06-17 报告，题为 From Pluggable to Co-Packaged: Scaling Silicon Photonics HVM for CPO。摘要明确 CPO HVM 挑战包括 known-good optical engines、scalable detachable fiber attachment、robust assembly, packaging and test solutions，以及 hyperscaler 对 cost、quality、reliability 的要求。
- 技术影响：CPO 从 ecosystem announcement 回到制造现实。量产重点会落在 optical engine KGD screening、fiber attach repeatability、laser/thermal drift、package-level optical alignment、ATE/SLT 覆盖率、RMA/field service 和 hyperscale volume ramp。
- 评级依据：A 级。官方 event page、关键词非常工程化，直接命中 CPO 量产；但这是会议预告。
- 可信度/待验证：需要等报告或后续 white paper 验证实际 HVM maturity、良率、客户验证、BER/pJ/bit 和 serviceability 数据。
- 对用户有什么用：后续 CPO 新闻先问四个问题：KGOE 怎么筛？fiber 怎么可拆可测？package test 覆盖什么？现场坏了怎么换？

### A 级｜Marvell/Luxshare 224G LR SerDes demo 与 co-packaged copper

- 日期：2026-05-28
- 来源：Marvell Blog
- 链接：https://www.marvell.com/blogs/224g-long-range-serdes-scale-up-scale-inside.html
- 事实摘要：Marvell 介绍其 3nm 224G long-range SerDes 与 Luxshare-Tech CPC/backplane/CPC channel 的 OFC 2026 演示，累计传输距离 2.5 m，目标场景包括 scale-up、scale-out 和 scale-inside AI fabrics。
- 技术影响：Copper 仍在和 optical I/O 竞争系统边界。SI/PI 难点包括 connector model、backplane loss、retimer/FEC、crosstalk、PDN noise coupling、thermal corner 和量产测试。
- 评级依据：A 级。头部 SerDes 厂商 + 具体链路演示 + 与 CPO/CPC 架构选择强相关。
- 可信度/待验证：需查看 channel loss profile、BERT 条件、FEC、jitter decomposition、temperature corner、多 lane 串扰和第三方互操作测试。
- 对用户有什么用：做光/铜取舍时，别只比较单 lane 速率；要比较 reach、connector count、serviceability、power/bit、FEC latency、thermal 和 field replace。

### A- 级｜UCIe 演进

- 日期：2026-06-02
- 来源：Semiconductor Engineering
- 链接：https://semiengineering.com/the-evolution-of-ucie/
- 事实摘要：文章讨论 UCIe 从 basic die-to-die connectivity 走向跨多 chiplet、不同封装方式和安全数据移动的互连标准。
- 技术影响：系统验证要覆盖 PHY margin、protocol stack、security state、flow control、latency determinism、power state transition、sideband/telemetry、package topology 和 compliance。

- 对用户有什么用：把 UCIe 项目评审从“PHY 能不能跑”升级为“协议、功耗状态、封装通道、SI/PI 和安全验证是否闭环”。
